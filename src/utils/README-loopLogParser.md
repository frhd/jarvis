# Loop Log Parser

A utility for parsing and analyzing `loop.log` files generated by the `loop.sh` script.

## Overview

The loop log parser extracts metrics from JSON streaming format logs, including:
- Token usage (input, output, cache read, cache write)
- Cost calculations based on Claude Opus 4.5 pricing
- Per-iteration breakdown
- Subagent (Task tool) statistics
- Progress tracking for long-running executions

## Features

### 1. Line-by-Line Parsing
Parse individual JSON log lines with graceful error handling for malformed entries.

```typescript
import { parseLoopLogLine } from '@utils/loopLogParser';

const line = '{"type":"loop_start_marker", "message":"New Claude Run 1"}';
const event = parseLoopLogLine(line);
// Returns: { type: 'loop_start_marker', message: '...' } or null
```

### 2. Complete Log Analysis
Parse entire log files to extract comprehensive metrics.

```typescript
import { parseLoopLog } from '@utils/loopLogParser';
import { readFile } from 'fs/promises';

const content = await readFile('./loop/loop.log', 'utf-8');
const metrics = parseLoopLog(content);

console.log(`Total Iterations: ${metrics.totalIterations}`);
console.log(`Total Cost: $${metrics.totalCost.toFixed(2)}`);
console.log(`Cache Hit Rate: ${(metrics.totalCacheRead / (metrics.totalTokensIn + metrics.totalCacheRead) * 100).toFixed(1)}%`);
```

### 3. Incremental Parsing
Read logs incrementally for real-time progress tracking (tail-like behavior).

```typescript
import { parseLoopLogIncremental } from '@utils/loopLogParser';

let lastLine = 0;

// Poll for updates every 5 seconds
setInterval(async () => {
  const { metrics, lastLine: newLastLine } = await parseLoopLogIncremental(
    './loop/loop.log',
    lastLine
  );
  lastLine = newLastLine;

  console.log(`Progress: ${metrics.totalIterations} iterations, $${metrics.totalCost.toFixed(2)}`);
}, 5000);
```

### 4. Subagent Statistics
Extract detailed statistics about Task tool invocations (subagents).

```typescript
import { extractSubagentStats } from '@utils/loopLogParser';

const stats = extractSubagentStats(logContent);

stats.forEach(agent => {
  console.log(`Agent ${agent.agentId} (${agent.model}):`);
  console.log(`  Tokens: ${agent.tokens.toLocaleString()}`);
  console.log(`  Duration: ${(agent.durationMs / 1000).toFixed(1)}s`);
  console.log(`  Tools: ${agent.toolCount}`);
});
```

## Types

The parser uses types from `src/types/plan.types.ts`:

### LoopLogEntry
```typescript
interface LoopLogEntry {
  iteration: number;
  timestamp: Date;
  tokensIn: number;
  tokensOut: number;
  cacheRead?: number;
  cacheWrite?: number;
  cost: number;
  duration?: number;
}
```

### LoopLogMetrics
```typescript
interface LoopLogMetrics {
  totalIterations: number;
  totalTokensIn: number;
  totalTokensOut: number;
  totalCacheRead: number;
  totalCacheWrite: number;
  totalCost: number;
  averageDuration?: number;
  entries: LoopLogEntry[];
}
```

## Log Format

The parser expects JSON streaming format with these event types:

### Loop Start Marker
```json
{"type":"loop_start_marker", "message":"==================== New Claude Run 1 ===================="}
```

### Message Start (contains model info)
```json
{"type":"stream_event","event":{"type":"message_start","message":{"model":"claude-opus-4-5-20251101",...}}}
```

### Message Delta (contains token usage)
```json
{
  "type":"stream_event",
  "event":{
    "type":"message_delta",
    "usage":{
      "input_tokens":100,
      "output_tokens":200,
      "cache_read_input_tokens":5000,
      "cache_creation_input_tokens":1000
    }
  }
}
```

### Task Tool Invocation (subagent)
```json
{
  "type":"assistant",
  "message":{
    "content":[{
      "type":"tool_use",
      "name":"Task",
      "id":"tool_123",
      "input":{"model":"sonnet"}
    }]
  }
}
```

### Task Result
```json
{
  "type":"user",
  "tool_use_result":{
    "agentId":"abc123",
    "totalTokens":5000,
    "totalDurationMs":3000,
    "totalToolUseCount":3
  },
  "message":{"content":[{"tool_use_id":"tool_123"}]}
}
```

## Pricing

The parser uses Claude Opus 4.5 pricing (as of January 2026):

- **Input tokens**: $15 per 1M tokens
- **Output tokens**: $75 per 1M tokens
- **Cache read**: $1.50 per 1M tokens (90% discount)
- **Cache write**: $18.75 per 1M tokens (25% premium)

Cost calculation:
```typescript
cost = (inputTokens / 1M * $15)
     + (outputTokens / 1M * $75)
     + (cacheReadTokens / 1M * $1.50)
     + (cacheWriteTokens / 1M * $18.75)
```

## Error Handling

The parser gracefully handles:
- **Malformed JSON**: Skipped with debug log entry
- **Missing files**: Returns empty metrics with error log
- **Invalid event types**: Ignored silently
- **Missing fields**: Uses default values (0 or empty arrays)
